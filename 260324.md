# Shell Script

Generelle Informationen:
- Shell Scripts haben die Dateiendung .sh
- Shell Scripts können in der Shell gestartet werden mit 'sh dateiname' oder './dateiname.sh'
    - Es werden allerdings execute Berechtigungen benötigt. Mit chmod +x dateiname hat man die execute Berechtigung.  

./ erscheint als komischer Syntax, allerdings deutet er an, in dieser Directory zu "suchen" und auszuführen. Wenn man ./ weglässt, will er die Datei in der $path Variable ausführen, in der sich die Datei allerdings nicht befindet.

In der Shell Datei wird alles wie üblich ausgeführt. Die gewohnten Shell Regeln bei Befehlen gelten hier auch, wie z.B.:
- Space (Leerzeichen) gilt immer als Abtrennung im Befehl
    - Aus diesem Grund strings mit Leerzeichen immer mit Double Quotes oder Single Quotes in Variablen speichern

# Variablen

Es können auch Variablen definiert werden, gekennzeichnet mit dateiname=...
aufgerufen werden sie wieder mit $

```sh
MYVAR="Hallo Welt"
echo $MYVAR
```

# Double und Single Quotes

In Double Quotes werden direkt die Variablen substituiert. In Single Quotes nicht, es wird wortwörtlich geschrieben.

Double Quotes:  
```sh
NAME="Christian Schuetzner"
TEXT="Hallo Welt $NAME"
```

Single Quotes:  
```sh
TEXT_2='Hallo $NAME'
```

# Curly Braces

Wenn ich z.B. $variablenname_xyz schreiben will, geht das nicht, da er nach dem \$ es als durchgehenden Variablennamen sieht. Hierfür kann man curly braces verwenden.
```sh
${FILE}_001.txt
```

Variablen anlegen:  
```sh

dir2="unterordner"
filename="meinedatei"
ext="txt"
```
Directories erzeugen:  
```sh
mkdir -p "$dir1/$dir2"
```

```
echo "hallo welt" > "$dir1/$dir2/$filename.$ext"
```

```sh
cat "$dir1/$dir2/$filename.$ext"
```
